Lorebook Process

LOREBOOK

Every creator eventually creates a world, and at some point wants to bind that world into a book. The Lorebook process compresses that work into a six-phase procedure.

It takes a 1,500-entry Vault as input and produces a B5 hardcover book of roughly 450 pages together with a public web edition. The same markdown source remains the single source of truth.

INPUT
Vault of 1,500 Entries or Collected IP Materials

An Obsidian Vault with structured frontmatter, or a partially organized body of IP materials ready to be mapped into a book structure.

→
OUTPUT
Print Edition + Public Web Edition

A B5 hardcover encyclopedia and a static website deployed from the same markdown source, with quarterly update cycles when needed.

Inside the Pipeline

Lorebook is an output stage of the Librarying Vault. Storytelling 100 branches from the same input.

↑ Input
Librarying Process

The standard input: 1,500 Vault entries mapped into a twelve-part matrix.
↔ Paired Output
Storytelling 100 Process

If Lorebook is the dictionary, Storytelling 100 is one hundred stories that use it.
↑ Paired Input
Seoyeongak Process

Translation assets become sources for multilingual publication.
↑ Paired Input
Character Process

Character sheets become the primary material for the character encyclopedia section.
